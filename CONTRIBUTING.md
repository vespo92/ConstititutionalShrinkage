# Contributing to Constitutional Shrinkage

Thank you for your interest in contributing to the Constitutional Shrinkage project! This is a grassroots movement to reimagine governance, and we need all the help we can get.

## How to Contribute

### For Developers

#### Getting Started
1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/ConstititutionalShrinkage.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature-name`

#### Development Workflow
```bash
# Run all apps in development mode
npm run dev

# Build all packages and apps
npm run build

# Run tests
npm run test

# Lint code
npm run lint

# Format code
npm run format
```

#### Code Standards
- Write TypeScript with strict type checking
- Follow the existing code style (enforced by ESLint and Prettier)
- Write tests for new features
- Document public APIs with JSDoc comments
- Keep functions small and focused
- Use meaningful variable names

#### Pull Request Process
1. Ensure your code passes all tests and linting
2. Update documentation if you've changed APIs
3. Write a clear PR description explaining your changes
4. Reference any related issues
5. Wait for review from maintainers

### For Designers

We need help with:
- UX/UI design for the citizen portal
- Data visualization for metrics dashboards
- Educational materials and infographics
- Brand identity and visual design

Submit design proposals as:
- Figma files
- Image files (PNG, SVG)
- Design documents

### For Policy Experts

We need help with:
- Constitutional framework refinement
- Legal compliance review
- Transition planning strategies
- Best practices from other democracies

Submit contributions as:
- Markdown documents in `/docs`
- Comments on existing proposals
- New policy proposals

### For Citizens

Everyone can contribute!
- Test the platform and report bugs
- Provide feedback on usability
- Share the project with others
- Participate in discussions
- Spread awareness on social media
